# 旺财 WAOS 满分审计标准

> 基于 ISO/IEC 25010 软件质量模型 + Google SRE PRR + Electron 生产最佳实践
> 每个维度 100 分制，85+ 通过，90+ 优秀，95+ 满分

## 8 大审计维度

### 维度 1: 功能完整性（Functional Suitability）
- 所有官称功能点 100% 可用（不只是能点开，要端到端跑通）
- 边界条件处理（空数据/非法输入/并发）
- 错误降级（API 不可用时有兜底）
- 功能闭环（每个功能有入口→执行→反馈→结果）

### 维度 2: 性能效率（Performance Efficiency）
- 首屏加载 < 3 秒
- API 响应 P95 < 500ms（AI 大脑除外）
- 内存占用 < 500MB（Electron 基线）
- 启动时间 < 5 秒
- 包体积合理（< 300MB）
- 无内存泄漏（长时间运行内存稳定）
- 大列表虚拟化（>100 条）

### 维度 3: 可靠性（Reliability）
- 异常自动恢复（进程崩溃→重启）
- 数据持久化（断电不丢）
- 网络中断容错（离线模式）
- 进程清理（退出时 kill 所有子进程）
- 端口冲突处理
- 数据库 WAL 模式 + 事务

### 维度 4: 安全性（Security）
- IPC contextBridge 隔离
- nodeIntegration: false
- 路径遍历防护
- Prompt 注入防护
- XSS/CSRF 防护
- 输入校验（所有 API 入参）
- 输出过滤（价格承诺/违规词）
- 敏感信息不泄漏（API Key 不进前端）
- 代码签名（Windows SmartScreen）

### 维度 5: 可维护性（Maintainability）
- TypeScript 严格类型（0 tsc 错误）
- ESLint 0 errors
- 模块化（单一职责）
- 代码复用（无重复代码）
- 中文注释完整
- 文档齐全（README/API/架构）
- 依赖更新（无已知漏洞）

### 维度 6: 可观测性（Operability）
- 结构化日志（console.log 带上下文）
- 错误追踪（try-catch + 错误堆栈）
- 性能指标（API 耗时/成功率）
- 健康检查端点
- 用户操作审计日志
- 实时监控面板

### 维度 7: 兼容性（Compatibility）
- Windows 10/11 64 位
- macOS（如需）
- Linux（开发环境）
- 浏览器降级（非 Electron 环境模拟模式）
- 屏幕分辨率适配（1366×768 ~ 4K）
- 深色/浅色/自动主题

### 维度 8: 用户体验（UX）
- 交互反馈（点击/加载/成功/失败）
- 响应式布局
- 无障碍（aria-label/键盘导航/对比度）
- 视觉一致性（颜色/间距/字号/圆角）
- 动画流畅（Framer Motion 60fps）
- 错误提示清晰可操作
- 空状态设计
- 加载骨架屏

## 评分标准

| 分数 | 等级 | 含义 |
|------|------|------|
| 95-100 | S | 满分，生产就绪 |
| 90-94 | A | 优秀，可发布 |
| 85-89 | B | 通过，小修可发布 |
| 80-84 | C | 需修复 |
| <80 | D | 不通过，严重问题 |

## 审计方法

1. **静态分析**：tsc + eslint + 代码审查
2. **动态测试**：端到端压测 + 边界用例
3. **运行时监控**：内存/CPU/日志
4. **安全扫描**：依赖漏洞 + 注入测试
5. **用户体验**：agent-browser 交互验证
6. **打包验证**：next build + electron-builder 产出检查
